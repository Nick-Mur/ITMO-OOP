# University — система управления курсами

## Условие лабораторной работы

Создать систему управления учебными курсами и преподавателями для университета с упором на наследование, полиморфизм и
работу с объектами. Система должна:

- добавлять и удалять курсы, назначать преподавателей и хранить записанных студентов;
- поддерживать онлайн- и офлайн-курсы с уникальными характеристиками;
- выдавать все курсы конкретного преподавателя;
- иметь юнит-тесты ключевой бизнес-логики на xUnit.
  (Паттерны Builder/Singleton рекомендованы как опция «со звёздочкой».)

## Как запустить

1. Установите .NET SDK 9.0+.
2. В корне репозитория выполните:
    - `dotnet restore`
    - `dotnet build University.sln`
3. Консольное приложение: `dotnet run --project University.ConsoleApp/University.ConsoleApp.csproj`
4. Юнит-тесты: `dotnet test University.Tests/University.Tests.csproj`

## Весь функционал для пользователя

Консольное меню `University.ConsoleApp` предоставляет:

1. Создание онлайн-курса (название, платформа, URL).
2. Создание офлайн-курса (название, корпус, аудитория).
3. Удаление курса по Guid.
4. Назначение преподавателя на выбранный курс (есть вывод справочников Id).
5. Запись студента на курс.
6. Просмотр всех курсов выбранного преподавателя.
7. Просмотр деталей курса (тип, назначенный преподаватель, список студентов).
   Приложение подготавливает демо-данные (по одному преподавателю, пару студентов и курсов) и отображает их Guid при
   старте, чтобы можно было сразу работать.

## Кратко об архитектуре

- Решение разделено на три проекта:
    - `University.Core` — доменная модель (`Person`, `Student`, `Teacher`), типы курсов (`Course`, `OnlineCourse`,
      `OfflineCourse`) и сервис управления курсами (`CourseServiceBase`, `CourseService`, `ICourseService`).
    - `University.ConsoleApp` — UI-слой, реализующий CLI поверх `ICourseService` и простых in-memory реестров
      преподавателей/студентов.
    - `University.Tests` — набор xUnit-тестов, покрывающий операции курсов и сервиса (создание, удаление, назначение
      преподавателя, зачисление студентов, выборка по преподавателю).
- Всё хранение выполняется в памяти (`Dictionary<Guid, ICourse>`), что упрощает расширение (можно заменить базовый
  сервис кастомной реализацией).
- Интерфейсы и базовые классы инкапсулируют общую логику; конкретные типы курсов добавляют специфичные свойства без
  дублирования кода.
